SnapMind Project Summary

AI 기반 스크린샷 및 사진 기록 관리 솔루션

프로젝트 개요

SnapMind는 사용자가 저장한 스크린샷과 사진을 AI(OCR, 이미지 분류)를 통해 자동으로 관리하고, 검색 및 메모 기능을 제공하여 "왜 저장했는지 잊어버리는" 문제를 해결하는 Android 애플리케이션입니다.

평가 항목(채점표) 만족 여부

본 프로젝트는 앱 개발 평가 항목을 충실히 반영하여 설계되었습니다.

  • ✔ 외부 APP 연동 / Intent 데이터 전달: 안드로이드 '공유(Share)' 기능(ACTION_SEND)을 통해 갤러리나 다른 앱에서 이미지를 SnapMind로 전달받아 저장합니다.
  • ✔ 머신러닝 (TensorFlow Lite): 기기 내장형(On-device) CNN 모델을 사용하여 이미지를 6~7개 카테고리(쇼핑, 코드, 밈 등)로 자동 분류합니다.
  • ✔ API 활용: OpenAI API 등을 활용하여 추출된 텍스트(OCR) 기반의 지능형 태그 추천 및 맥락 파악 기능을 제공합니다. (또는 YouTube API 연동)
  • ✔ DB 활용 및 Jetpack Library (3개 이상): Room Database를 통해 메타데이터를 저장하며, UI 구현에 ViewPager2, DrawerLayout, RecyclerView, Fragment, ViewModel, Navigation Component 등 다수의 Jetpack 라이브러리를 적극 활용합니다.
  • ✔ Coroutine 활용: UI 스레드를 차단하지 않기 위해 OCR 처리, 이미지 ML 분류, DB 저장 및 API 요청 등 모든 무거운 작업은 Coroutine을 통한 비동기식(Background)으로 처리됩니다.
  • ✔ 다운로드 매니저 / 이미지 로딩: 외부 API 이미지나 기기 내 썸네일을 효율적으로 표시하기 위해 Glide(혹은 시스템 다운로드 매니저 연동)를 사용합니다.

핵심 기능 및 세부 동작 로직

앱의 주요 파이프라인은 사용자가 이미지를 추가한 직후 백그라운드에서 자동으로 진행됩니다.

1. 이미지 가져오기 (Ingestion)

로직: 사용자가 외부 앱에서 공유(Share) 버튼을 누르면 SnapMind의 AddScreenshotActivity(또는 Intent Receiver)가 트리거됩니다. 앱은 이미지의 URI를 확보하고, 즉시 Room DB에 임시 MemoryItem 레코드를 생성하여 상태를 '처리 중'으로 표시합니다.

2. OCR 텍스트 추출 (Text Recognition)

로직: Coroutine 백그라운드 스레드에서 ML Kit OCR 프로세서가 실행됩니다. 이미지 내부의 텍스트(예: 에러 로그, 유튜브 제목, 쇼핑몰 상품명)를 스캔하여 텍스트 블록으로 추출한 뒤, DB의 해당 레코드에 텍스트 데이터를 병합(Update)합니다.

3. AI 이미지 분류 (Classification)

로직: OCR과 병렬적(혹은 순차적)으로 TensorFlow Lite 모델이 이미지를 분석합니다. 이미지가 '코드(Code)' 화면인지 '쇼핑(Shopping)' 화면인지 확률(Confidence)을 계산하여 가장 높은 카테고리를 DB에 저장합니다.

4. 자동 태깅 및 데이터 정제 (Auto-Tagging)

로직: 추출된 OCR 텍스트와 ML 카테고리를 기반으로 키워드를 추출합니다. (예: OCR에 "NullPointerException"이 있고 카테고리가 "Code"이면 #버그, #개발 태그 자동 할당). 필요시 OpenAI API를 호출하여 보다 정교한 태그를 생성한 뒤 최종 상태를 DB에 확정(Completed) 짓습니다.

핵심 UI 컴포넌트 명세 (Jetpack)

프로젝트 전반에 걸쳐 사용되는 핵심 Android UI 라이브러리 및 적용 방안입니다.

1. ViewPager2

역할: 앱의 최상위 루트 탐색(App Shell) 및 페이지 간의 매끄러운 스와이프 전환 제공.

예시: 사용자가 '홈(Memory)' 탭에서 화면을 왼쪽으로 밀면 자연스럽게 '검색(Search)' 탭으로 전환됩니다. BottomNavigationView와 완벽하게 동기화됩니다.

2. DrawerLayout

역할: 빠르고 직관적인 카테고리 및 태그 필터링(Quick Filters) 메뉴 제공.

예시: 메인 홈 화면 좌측 가장자리를 스와이프하면 서랍 메뉴가 열립니다. 수많은 스크린샷 중 [Shopping] 카테고리를 터치하면 즉시 리스트가 쇼핑 관련 이미지로만 필터링됩니다.

3. RecyclerView

역할: 방대한 양의 스크린샷, 메모, 검색 결과 목록을 메모리 효율적으로 렌더링.

예시: 격자형(Grid) 이미지 갤러리 뷰, 검색 화면의 세로 리스트, 가로로 스크롤되는 추천 태그 목록 등에 ListAdapter와 함께 광범위하게 사용됩니다.

4. Fragment

역할: 재사용 가능하고 독립적인 UI 생명주기 관리 모듈.

예시: ViewPager2 내부의 각 화면(MemoryFragment, SearchFragment 등)을 독립적인 뷰 컨테이너로 캡슐화하여 MVVM 패턴을 깔끔하게 유지합니다.

로드맵 및 세부 항목 (마일스톤)

개발은 단계별로 진행되며, 각 마일스톤 완료 시 특정 기능을 사용할 수 있게 됩니다.

M0: Project Foundation (기반 설정)

Not Started
  • Hilt 의존성 주입(DI) 설정 및 앱 패키지 구조화
  • Room Database 스키마 뼈대 작성
  • ViewPager2 + BottomNavigationView를 이용한 메인 앱 셸(App Shell) 구축
  • DrawerLayout을 이용한 퀵 필터 메뉴 기반 마련

M1: Capture and Import (수집 및 저장)

Not Started
  • Android ACTION_SEND 인텐트를 통한 외부 공유 이미지 수신 기능 구현
  • 수신한 이미지의 디바이스 로컬 저장소 캐싱 및 File URI 생성
  • Room DB에 MemoryItem 기본 정보(날짜, URI) 저장 로직 완성

M2: OCR Pipeline (텍스트 추출)

Not Started
  • Google ML Kit 의존성 추가 및 비전 텍스트 인식 프로세서 구현
  • 코루틴(Coroutine) 백그라운드를 활용한 비차단 텍스트 추출
  • 추출된 텍스트를 Room DB에 텍스트 인덱스로 저장 (검색 대비)

M3: AI Classification (이미지 분류)

Not Started
  • 사전 학습된 TFLite 모델(.tflite) 앱 리소스에 탑재 및 인터프리터 설정
  • 저장된 이미지를 모델 입력 크기에 맞춰 리사이징 및 정규화
  • 추론 결과(Category 및 Confidence Score)를 파싱하여 DB에 업데이트

M4: Auto-Tagging (자동 태깅)

Not Started
  • OCR 텍스트 내 특정 키워드(예: 'Exception', '₩', '결제') 매칭 룰 엔진 구축
  • OpenAI API 연동을 통한 자연어 기반 동적 태그 생성 기능 탑재
  • 생성된 태그를 메모와 다대다(N:M) 관계로 DB에 맵핑

M5: Search and Filter (검색 및 필터)

Not Started
  • 검색어(Query)를 기반으로 OCR 텍스트, 메모, 태그 통합 검색 쿼리 작성 (Room FTS 활용 고려)
  • DrawerLayout 서랍 메뉴를 통한 카테고리(Code, Shopping 등) 전역 필터링 적용
  • 검색 결과를 RecyclerView에 실시간 갱신(LiveData/Flow)

M6~M7: Memo Management & Stabilization (메모 관리 및 안정화)

Not Started
  • 사용자 직접 텍스트 메모 작성, 수정, 삭제, 보관함 이동 기능 추가
  • 앱 크래시 방어, 코루틴 예외 처리, Room 마이그레이션 테스트
  • UI/UX 애니메이션 폴리싱 및 성능 최적화

기술 스택 요약

Kotlin Coroutines Room (DB) Retrofit (API) ML Kit (OCR) TensorFlow Lite Hilt (DI) Navigation Component Glide (Image)
